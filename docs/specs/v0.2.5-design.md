# v0.2.5 Design — 대화 기억 + 개인 맞춤 + 구글 검색

**Date**: 2026-03-13
**Features**: F-02 (지난 대화 기억), F-03 (오늘 기반 질문 → F-30 흡수), F-04 (개인 기반 질문), F-30 (대화 중 구글 검색)

## Summary

대화 종료 시 요약을 자동 생성하여 다음 대화의 시스템 프롬프트에 주입하고,
Soul 프로필의 관심사를 활용한 토픽 제안 + Gemini googleSearch 도구로 실시간 검색을 지원한다.

## Data Model

### Conversation 엔티티 — 컬럼 추가

```typescript
@Column({ type: 'text', nullable: true })
summary: string | null
// "손자 민수 운동회 이야기, 무릎 통증으로 못 갈까 걱정, 내일 병원 예약"
```

- 1~3문장 텍스트 요약
- 대화 미종료 또는 요약 실패 시 `null`
- SoulProfile, transcript 등 기존 구조 변경 없음

## Data Flow

### 대화 시작 시

```
클라이언트 연결
  → soulService.getProfileText(deviceId)                    // 기존 — Soul 프로필
  → conversationService.getRecentSummaries(deviceId, 3)     // 새로 — 최근 3회 요약
  → buildSystemPrompt(soulContext, recentSummaries)
  → Gemini Live 세션 생성
```

### 대화 종료 시

```
대화 종료
  → conversationService.end(id, transcript)                 // 기존
  → 병렬 fire-and-forget:
      1) soulService.extract(deviceId, convId, transcript)  // 기존 — Soul 프로필 업데이트
      2) conversationService.summarize(convId, transcript)  // 새로 — 요약 생성
```

### 요약 생성

- Gemini Text API 호출 (`responseMimeType: 'application/json'`)
- 입력: transcript 텍스트 (8000자 제한, 기존 Soul과 동일 패턴)
- 출력: `{ "summary": "..." }`
- ConversationService에 메서드 추가, GeminiModule 의존성 필요

## Error Handling

| 실패 지점 | 처리 |
|-----------|------|
| 요약 생성 Gemini 호출 실패 | `summary` NULL 유지. 다음 대화에 요약 없이 진행 |
| 최근 요약 조회 실패 | Soul 프로필만으로 시스템 프롬프트 구성 |
| Soul 추출 실패 | 요약은 독립 저장됨. 서로 영향 없음 |

모두 fire-and-forget — 대화 종료 응답을 블로킹하지 않음.
