# hallmai v0.1.0 MVP Spec
> 사용자 관점의 최소 제품 기획서
> 작성일: 2026-03-07

---

## 한 줄 정의

**시니어의 일상을 함께하는 AI 친구. 가족은 그 온기가 자연스럽게 느껴집니다.**

---

## 제품 철학

AI 기술은 앞선 세대만의 것이어서는 안 됩니다.

---

## 왜 이 제품이 필요한가

| 문제 | 할마이의 해결 |
|---|---|
| 어르신은 외롭지만 먼저 연락하기 미안하다 | AI 친구가 먼저 말을 걸어준다 |
| 혼자 있는 시간이 길고 말동무가 없다 | 매일 일상을 함께하는 친구가 생긴다 |
| 가족은 전화해도 "잘 있어~" 한 마디로 끝난다 | 어르신이 오늘 무슨 얘기를 했는지 자연스럽게 알게 된다 |
| 바빠서 자주 전화 못 하는 게 죄책감이 된다 | 짧아도 맥락 있는 대화가 가능해진다 |

## 페르소나

### 시니어 — 박순자 (72세)

혼자 사는 독거 어르신. 카카오톡으로 가족 사진 받아보고, 유튜브로 트로트 찾아보는 정도. 앱 설치나 새로운 건 가족이 도와줘야 함. 낮에 혼자 있는 시간이 길고, 먼저 전화하는 걸 "폐 끼친다"고 생각함. 말동무가 생기면 좋겠지만 딱히 방법이 없었음.

### 가족 — 박민준 (42세, 아들)

바쁜 직장인. 어머니가 걱정되지만 매일 전화는 부담. 전화해도 "잘 있어~" 한 마디로 끝나서 실제로 어떻게 사시는지 잘 모름. 오늘 무슨 얘기를 하셨는지 알면 전화할 때 훨씬 편할 것 같다고 생각함.

## 사용자 시나리오

### 시나리오 1: 어머니의 하루 (시니어)

```
1. 어머니가 앱을 열면 바로 통화 화면이 보인다
2. 화면 가운데 큰 버튼을 누른다
3. AI가 "안녕하세요, 반가워요!" 하고 먼저 인사한다
4. 어머니가 오늘 있었던 일을 이야기한다
5. AI가 듣고, 공감하고, 가끔 질문도 한다
6. 대화가 끝나면 AI가 "좋은 하루 보내세요!" 하고 인사한다
```

### 시나리오 2: 아들의 하루 (가족)

```
1. 자녀가 앱을 열면 홈 화면이 보인다
2. 어머니가 나눈 이야기 카드가 보인다
3. "이웃이랑 산책 다녀온 얘기를 하셨어요" 같은 요약이 있다
4. 어머니의 한마디가 카드에 담겨있다 — "봄이 오니까 괜히 기분이 좋아"
5. 자녀가 카드를 읽고 어머니에게 전화를 건다
```

## 화면 & 기능 (v0.1.0)

앱 진입 시 기본 모드는 시니어 모드. 가족은 설정에서 로그인 후 전환.

v0.1.0 배포 방식: iOS — TestFlight, Android — APK 직접 설치.

### 시니어 플로우 (비로그인)

앱 최초 실행 시 기기 단위 UUID가 자동 발급되며, 이것이 사용자 코드로 사용됨.

시니어 온보딩은 별도 화면 없이 진행. 가족이 설치해주며 설명하거나, 시니어 간 바이럴로 혼자 쓰다가 가족이 나중에 연결하는 두 가지 진입 경로를 전제로 함.

| 화면 | 기능 | 핵심 요소 |
|---|---|---|
| 통화 화면 | 원버튼 음성 통화, 실시간 음성 대화 (Gemini Live API, 인터럽트 지원) | 큰 버튼 하나, 상태 표시 (대기/듣는중/말하는중/종료), 통화 중 종료 버튼 노출, 좌측 상단 사용자 코드 항상 노출, 우측 상단 설정 버튼 |
| 설정 화면 | 가족 연결 | 구글 로그인 진입점 |

### 가족 플로우 (로그인 후)

| 화면 | 기능 | 핵심 요소 |
|---|---|---|
| 가입/로그인 화면 | 구글 로그인 | 구글 로그인 버튼 |
| 코드 입력 화면 | 시니어 연결 | 코드 입력 필드, 코드 위치 안내 텍스트 |
| 홈 화면 | 이야기 카드 열람 (읽기 전용) | 카드 타임라인, 빈 상태 처리 |
| 설정 화면 | 시니어 관리 | 연결된 시니어 정보, 연결 해제 |

## 오늘의 이야기 카드 스펙

카드는 확장 가능한 구조로 설계. v0.1.0은 `daily_story` 타입 하나만 구현.

하루에 카드 한 장 생성. 전날 대화 전체를 합산해서 다음날 오전 10시에 생성됨. 전날 대화가 없으면 카드 생성 안 됨.

**공통 필드**

| 필드 | 타입 | 설명 |
|---|---|---|
| type | string | 카드 종류 |
| date | datetime | 생성 일시 |
| data | object | 카드 종류별 데이터 |

**v0.1.0 카드 타입: `daily_story`**

| 필드 | 타입 | 예시 |
|---|---|---|
| topic | string | "이웃이랑 산책 다녀온 얘기" |
| quote | string | "봄이 오니까 괜히 기분이 좋아" |
| vibe | enum | warm / calm / quiet |

**카드 작성 원칙**
- 건강 세부 내용 (구체적 증상, 약 이름) → 포함 금지
- 가족 갈등 → 포함 금지
- 금전 관련 발화 → 포함 금지

---

## 구현 완료 기준 (v0.1.0)

1. **어르신이 버튼을 눌러 AI와 실제 음성 대화를 할 수 있다**
2. **전날 대화가 끝나면 다음날 오전 10시에 카드가 자동 생성된다**
3. **가족이 홈 화면에서 이야기 카드를 볼 수 있다**

## 피쳐 후보 목록 (v0.1.0 이후)

| # | 기능 | 설명 |
|---|---|---|
| F-01 | AI 선제 대화 시작 | 매일 정해진 시간에 AI가 먼저 시니어에게 말을 걸어 대화 시작 |
| F-02 | AI 질문 풀 — 오늘 기반 | 날씨, 계절, 시사 기반으로 AI가 꺼낼 대화 주제 자동 생성 |
| F-03 | AI 질문 풀 — 개인 기반 | 가족이 입력한 어르신 취미/관심사 기반으로 대화 주제 생성 |
| F-04 | AI 질문 풀 — 관계 기반 | 가족 구성원 정보를 바탕으로 가족 얘기를 자연스럽게 꺼내는 대화 주제 생성 |
| F-05 | 지난 대화 기억 | 이전 대화 내용을 기억해서 다음 대화에 이어가기 |
| F-06 | 가족 온보딩 | 어르신 취미, 가족 구성원 정보를 가족이 입력하는 온보딩 화면 |
| F-07 | 음성 브리핑 | 어제의 이야기 카드 요약을 TTS로 가족에게 들려주기 |
| F-08 | Soul 설정 | AI 호칭, 인사말, 대화 톤 등 가족이 커스터마이징 |
| F-09 | 주간 vibe 차트 | 7일간 vibe 변화를 가족 홈 화면에 시각화 |
| F-10 | 주간 인사이트 | AI가 한 주 대화를 분석해 가족에게 요약 카드 제공 |
| F-11 | 다중 가족 구성원 | 여러 가족이 각자 계정으로 같은 시니어의 홈 화면 확인 |
| F-12 | 가족 답장/사진 전송 | 가족이 홈 화면에서 어르신에게 메시지/사진 보내기 |
| F-13 | 앱스토어 배포 | iOS App Store / Android Play Store 정식 배포 |
| F-14 | 소리 기반 자동 연결 | 시니어와 가족이 함께 있을 때, 가족 폰에서 소리 송출 → 시니어 폰이 감지 → 자동 연결. 코드 입력 대체가 아닌 추가 연결 방식 |
| F-15 | 기기 변경 시 프로필 이전 | 시니어가 기기를 바꾸거나 앱을 재설치해도 기존 UUID 기반 연결 유지 |
| F-16 | 대화 종료 트리거 개선 | 침묵 감지 자동 종료, AI 판단 종료 등 다양한 종료 방식 지원 |
| F-17 | vibe enum 개선 | 부정 감정을 포함한 vibe 스펙트럼 재설계 |
| F-18 | 시니어 앱 연결 가족 확인/해제 | 시니어 앱 설정에서 연결된 가족 목록 확인 및 해제 가능 |
