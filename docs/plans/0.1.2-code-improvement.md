# v0.1.2 Code Improvement Plan

> 0.1.1 코드리뷰 기반 코드 품질/보안 개선

## 목표

0.1.1 코드리뷰에서 발견된 Critical 13건, Major 25건을 우선순위별로 수정하여

## Phase 1: 보안 수정 (Critical)

### BE-SEC-1: JWT fallback secret 제거
- **파일**: `backend/src/modules/auth/jwt.strategy.ts:13`
- **문제**: `'fallback-secret'` 하드코딩 → 토큰 위조 가능
- **수정**: fallback 제거, `JWT_SECRET` 미설정 시 앱 시작 차단

### BE-SEC-2: CORS origin 제한
- **파일**: `backend/src/main.ts:17`
- **문제**: `enableCors()` 무제한 → 모든 오리진 허용
- **수정**: `CORS_ORIGIN` 환경변수 기반 화이트리스트

### BE-SEC-3: WebSocket 인증 추가
- **파일**: `backend/src/modules/voice/voice.gateway.ts`
- **문제**: 인증 없이 누구나 음성 세션 시작 가능
- **수정**: `handleConnection`에서 JWT 토큰 검증

### BE-SEC-4: crypto.randomInt 교체
- **파일**: `backend/src/modules/device/device.service.ts:117`
- **문제**: `Math.random()` → 예측 가능한 디바이스 코드
- **수정**: `crypto.randomInt()` 사용

### BE-SEC-5: synchronize 환경 제한
- **파일**: `backend/src/common/entity/entity.module.ts:20`
- **문제**: 프로덕션에서 `synchronize: true` → 스키마 자동 변경
- **수정**: `NODE_ENV !== 'production'` 조건 추가

### BE-SEC-6: 에러 메시지 노출 차단
- **파일**: `backend/src/common/response/error.filter.ts:40-41`
- **문제**: 내부 에러 메시지가 클라이언트에 직접 노출
- **수정**: 프로덕션에서는 generic 메시지 반환

### FE-SEC-1: Google Client ID fallback 제거
- **파일**: `frontend/src/app/providers.tsx:7-8`
- **수정**: 환경변수 미설정 시 빈 문자열 또는 에러

### FE-SEC-2: userScalable 허용
- **파일**: `frontend/src/app/layout.tsx:19-20`
- **문제**: 고령자 대상 서비스에서 핀치 줌 비활성화 (접근성 위반)
- **수정**: `userScalable: false` 제거
