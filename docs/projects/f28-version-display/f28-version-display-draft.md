# F-28 설정 화면 버전 정보 표시 Draft

---

## 왜 만드나

QA·CS 과정에서 "지금 어떤 버전이에요?"라는 질문이 반복된다.
앱스토어 배포 이후에는 앱과 서버 버전이 다를 수 있어, 설정 화면에서 즉시 확인할 수단이 필요함.

---

## 화면 구성

#### 설정 화면 하단 — 버전 정보 영역

약관 링크 아래, 화면 최하단에 버전 정보를 텍스트로만 표시.
별도 카드 없이, 터치도 되지 않는 정보 전용 영역.
서버에 연결이 안 될 경우 서버 버전은 `—`로 표시되고 앱은 정상 작동함.

**표시 방식에 대해 두 가지 옵션을 검토 중이다.**

---

**Option A — 항목 분리형**

앱과 서버를 각각 한 줄씩 명시적으로 표시.

```
앱    v0.2.0
서버  v0.2.0
```

QA·CS 응대에서 버전 불일치를 가장 빠르게 파악할 수 있음.
다만 일반 사용자 입장에서 "서버 버전"이라는 개념이 낯설게 느껴질 수 있음.

---

**Option B — 압축 단일행형**

한 줄에 두 버전을 나란히 표기해 화면 면적을 최소화.

```
버전  v0.2.0 (앱)  v0.2.0 (서버)
```

또는 기호로만 구분:

```
버전  v0.2.0 · v0.2.0
```

일반 사용자 눈에 덜 띄어 기술 정보를 자연스럽게 뒤로 물릴 수 있음.
대신 `·` 기호만 사용할 경우, 어느 쪽이 앱·서버인지 별도 맥락 없이는 구분하기 어려움.

---

**두 옵션 모두 고려할 만하다.** 현재 단계(MVP, QA 빈도 높음)에서는 Option A가 실용적이며, 정식 출시 이후 일반 사용자 비중이 커지면 Option B로 전환을 검토할 수 있음 (추후 변경 가능).

---

## 유저 흐름

**① QA·CS 담당자**
설정 진입 → 화면 하단 스크롤 → 앱/서버 버전 확인 → 이슈 리포트에 기재

**② 일반 사용자 (로그인)**
설정 진입 → 프로필·약관 확인 → 로그아웃 → 버전 영역은 인지하지 않고 지나침

**③ 일반 사용자 (비로그인)**
설정 진입 → 로그인 안내·약관 확인 → 버전 영역은 인지하지 않고 지나침

---

## 서비스 변경 범위

**새로 추가**
- 버전 정보 영역 — 설정 화면 최하단, 로그인/비로그인 공통
- 서버 버전 조회 — 헬스체크 응답에 버전 포함 (기존 `GET /api/health` 확장)

**달라지는 것**
- 설정 화면 (`settings/page.tsx`) — `LegalLinks` 아래에 버전 컴포넌트 추가
